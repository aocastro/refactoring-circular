## 2025-04-06 - Linktree XSS Vulnerability
**Vulnerability:** Cross-Site Scripting (XSS) via unvalidated user-supplied URLs in the Linktree functionality. User-provided `link.url` was directly interpolated into `href` attributes in anchor tags (`<a href={link.url}>`) and `window.open(link.url)` calls in `src/components/dashboard/LinktreeContent.tsx` and `src/pages/LinktreePublic.tsx`.
**Learning:** The application allowed users to create "Linktree" links with custom URLs, but did not restrict the URI scheme. This allowed an attacker to input a URL like `javascript:alert('XSS')`, which would execute arbitrary JavaScript in the context of the user viewing the Linktree page when clicked. The architecture lacked a central, reusable utility for URL sanitization.
**Prevention:** Always sanitize user-provided URLs before rendering them in `href` or passing them to `window.open`. Created a `sanitizeUrl` utility in `src/lib/utils.ts` to explicitly block dangerous schemes (`javascript:`, `vbscript:`, `data:`) and fallback to `about:blank`. This utility must be consistently applied across the codebase wherever external URLs are rendered.
